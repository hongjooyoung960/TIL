# 데이터 모델

PostgreSQL 테이블과 각 테이블이 존재하는 이유를 요약합니다.

## `daily_plans`

하루 단위 계획의 루트 레코드입니다. 기상·취침 시간, 그날의 메인 목표, 메모, 그날 활동 전체로부터 계산된 성취 지표를 보관합니다.

- **`plan_date`**: 자연일 기준 고유 키로 하루에 하나의 플랜만 허용합니다.
- **`achievement_rate`**: 활동의 중요도 가중 성취율(주 지표).
- **`total_score`**: 완료 활동 수 기준 단순 성취율과 같은 역할을 하는 보조 지표로 사용합니다.

## `activities`

특정 일간 플랜에 속한 작업·루틴·건강 등 실행 단위입니다. 상태(`planned`, `done`, `missed`, `partial`)와 중요도·집중도 점수가 통계 및 차트의 입력이 됩니다.

- **`daily_plan_id`**: 플랜 삭제 시 활동도 함께 제거되도록 CASCADE를 사용합니다.

## `goals`

장기·월간·주간·일간 목표를 단일 테이블로 표현합니다. `parent_goal_id`로 상위 목표와 연결해 계층을 표현합니다.

- **`progress`**: UI에서 진행률 퍼센트를 표시합니다(MVP에서는 사용자 입력 및 간단한 표시 중심).

## `daily_goal_links`

그날의 플랜이 어떤 상위 목표에 얼마나 기여했는지 연결합니다. 하루 플랜과 목표가 다대다에 가까운 관계일 때 중간 테이블이 필요합니다.

- **`contribution_score`**: 그날 해당 목표에 대한 기여도(1–5).

## `git_commit_logs`

일간 산출물(JSON/Markdown)을 Git으로 올릴 때마다 성공·실패와 커밋 해시·오류 메시지를 남겨 UI에서 최근 배포 상태를 확인할 수 있게 합니다.
